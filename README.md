## Projects

### [VaniaBot](https://github.com/CARLOSGRCIAGRCIA/VaniaBot)
WhatsApp automation system — 90+ commands, 60+ services, up to 50 concurrent sessions. The interesting part is the architecture: middleware pipeline + command registry so adding features never touches core logic, a Circuit Breaker fallback chain across multiple AI providers, and a SubBot orchestrator with automatic recovery when instances go down.
`TypeScript` `Node.js` `Redis` `Bull` `SQLite` `Vitest` `Zod` `Docker`

### [Commons Marketplace](https://github.com/CARLOSGRCIAGRCIA/commons-marketplace)
Full-stack e-commerce platform deployed to production. Implemented CI/CD with Blue/Green deployment strategy for zero-downtime releases. Integrated Supabase for auth and Ably for real-time messaging.
`JavaScript` `Node.js` `Express` `Supabase` `Ably` `CI/CD`

### [Pathfinder API](https://github.com/CARLOSGRCIAGRCIA/pathfinder)
REST API for optimal route calculation using A* with multi-waypoint support. Full unit and E2E test coverage with Jest, centralized error handling and structured logging.
`Node.js` `JavaScript` `Jest`

### [Distributed Payment System](https://github.com/CARLOSGRCIAGRCIA)
Decoupled architecture across two services: JWT authentication in Java/Spring Boot and business logic in C#/.NET with cross-service token validation. Centralized auth middleware.
`Java` `Spring Boot` `C#` `.NET` `JWT` `MySQL`
